这份PPT是斯坦福大学2025年秋季课程**CS146S《现代软件开发人员》（The Modern Software Developer）**的一份讲义，由Mihail Eric授课。本次课程的主题是**“从零开始构建编程智能体（Building a Coding Agent From Scratch）”**。

以下是本次课程内容的详细总结：

### 1. 核心架构与基本术语
课程首先介绍了构建智能体时涉及的核心概念和角色定义：
*   **交互模型：** 编程智能体的工作流程涉及**用户、智能体接口以及大语言模型（LLM）**之间的循环交互，并配合特定的**工具（Tools）**来完成任务。
*   **三大提示词类型：**
    *   **系统提示词（System prompt）：** 用于定义大模型的整体行为准则和指令。
    *   **用户提示词（User prompt）：** 用户提出的具体自定义请求。
    *   **助手提示词（Assistant prompt）：** 大语言模型生成的响应内容。

### 2. 构建编程智能体的具体步骤
要实现一个能够自动编写代码的智能体，需要遵循以下流程：
*   **对话管理：** 从终端读取输入，并将内容持续附加（append）到对话历史中。
*   **工具集成：** 明确告知LLM有哪些可用工具，例如 `Read_file`（读取文件）、`List_dir`（列出目录）和 `Edit_file`（编辑文件）。
*   **执行循环：** LLM会在合适的时间请求调用工具；开发者或系统在离线状态下执行这些工具，并将执行结果返回给模型。
*   **文件操作：** 智能体最终应具备创建和编辑新文件的能力。

### 3. 高阶优化技巧（“秘密配方”）
讲义深入探讨了类似 Claude 这样成熟产品的幕后技术，即如何提升智能体的性能：
*   **上下文管理：** 
    *   通过**预载入（Front-load）**微小且有针对性的提示词来提供背景。
    *   使用**子智能体（Sub agents）**来处理任务，以防止上下文过载（Context overloading）。
*   **防止指令偏移（Drift）：** 在系统/用户提示词、工具调用及结果中到处嵌入**系统提醒（System reminders）**，并使用如 `<system-reminder>` 之类的标签来确保模型始终遵循既定规则。
*   **命令提取：** 采用**命令前缀提取（Command prefix extraction）**技术来更精确地识别指令。

这份讲义强调了通过简单的工具调用和严密的提示词工程，即可从零开始搭建出一个具备实用功能的编程机器人。
