## 作业概述
在 3 个不同的技术栈中构建相同功能的 Web 应用程序。至少一个版本必须使用 [`bolt.new`](https://bolt.new/)（一个 AI 应用生成平台）创建。至少一个版本必须使用非 JavaScript 语言作为前端或后端（例如，Django、Ruby on Rails）。

你可以重用之前几周的应用程序（"开发者控制中心"）或创建你选择的新应用程序，只要它满足 [最小功能范围](#minimum-functional-scope)。应用程序应该是端到端功能的（前端 + 后端 + 适当时的持久性），并展示连贯的功能集。

## 最小功能范围
- 用户可以创建、读取、更新和删除主要资源（例如，笔记、任务、帖子）。
- 持久存储（数据库或基于文件的），适用于堆栈。
- 基本验证和错误处理。
- 简单但功能齐全的 UI，展示主要流程。
- 清晰的说明，可在本地运行每个版本（如果部署，则提供部署链接）。

## 堆栈要求
构建同一应用程序的 3 个独立版本，每个版本使用不同的堆栈。示例：
- MERN（MongoDB、Express、React、Node.js）
- MEVN（MongoDB、Express、Vue.js、Node.js）
- Django + React（或 Vue）
- Flask + Vanilla JS（或 React）
- Next.js + Node（或 NestJS）
- Ruby on Rails（全栈）

提醒：至少一个版本必须包含非 JavaScript 语言作为前端或后端（例如，Python/Django、Ruby/Rails）。


至少一个版本必须使用 AI 应用生成平台 **[`bolt.new`](https://bolt.new/)** 构建，但你可以为其他版本探索其他应用生成平台（例如 Lovable、Figma Make）。


## 了解 Bolt
Bolt 是一个 AI 辅助开发平台，可从自然语言提示生成网站、Web 应用和移动应用。用户可以用纯文本描述他们的想法，Bolt 在几分钟内生成功能原型——从登录页面和电子商务网站到 CRM 和移动工具。[在此处](https://support.bolt.new/building/intro-bolt) 了解更多信息。

## 使用 AI 应用生成器的提示
- 像 Bolt 这样的应用生成器最适合现代全栈技术，当你使用它们而不指定特定框架时，你将默认获得这些技术。
- 更喜欢从描述你的应用概念、实体、路由和 UI 流程的干净提示开始。
- 在提示中清楚地描述数据模型和关系。
- 迭代完善数据模型、CRUD 端点、认证（如果使用）和前端组件的提示。
- 保持每个版本隔离，以避免依赖冲突。
- 导出或同步生成的代码，并将其作为该堆栈的独立项目文件夹提交。

## 交付物
