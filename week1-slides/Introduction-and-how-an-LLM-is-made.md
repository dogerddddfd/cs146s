这份PPT源自斯坦福大学2025年秋季课程**CS146S《现代软件开发人员》（The Modern Software Developer）**，由Mihail Eric授课。该课程旨在探讨在AI技术飞速发展的背景下，软件工程师应如何转型与生存。

以下是该PPT的核心内容总结：

### 1. 行业现状与挑战（2025年的世界）
*   **AI革命：** 生成式AI正引领一场职业革命，AI创业公司估值飙升。
*   **挑战（坏消息）：** 调查显示AI编写代码的比例极高（Windsurf团队称约95%的代码由其Agent编写；Anthropic CEO预测AI将在数月内完成90%的代码工作），谷歌首席科学家预测AI一年内可达到初级程序员水平。
*   **机遇（好消息）：** 开发人员的**生产力将达到史上最高**。**你不会被AI取代，但会被擅长使用AI的胜任工程师取代**。

### 2. 现代软件开发人员的核心素养
*   **人机协作工程（Human-agent engineering）：** 重点培养AI系统尚无法替代的技能，包括**理解业务逻辑**和**担任技术负责人（Tech Lead）**。
*   **深度理解：** LLM的表现取决于上下文质量，如果你不能理解自己的代码库，AI也无法理解。
*   **行动指南：**
    *   **大量阅读与审查代码：** 培养辨别软件优劣的“品味”。
    *   **积极实验：** 目前尚未形成固定的软件模式，需要不断尝试各种工作流和工具，找到最适合自己的方式。

### 3. 大语言模型（LLM）的技术原理
*   **基本原理：** LLM是用于**预测下一个token**的自回归模型。其过程包括：输入分词（Tokenize） -> 嵌入层（Embedding） -> 包含自注意力机制的Transformer层 -> 输出概率分布。
*   **训练三个阶段：**
    1.  **自我监督预训练（Pretraining）：** 在数万亿token上学习语言和代码的基础知识。
    2.  **监督微调（SFT）：** 学习遵循指令（如：写一个for循环）。
    3.  **偏好对齐（Preference tuning）：** 根据人类的有用性、准确性和可读性偏好进行调整。
*   **进阶模型：** 推理模型（Reasoning models）通过**思维链（CoT）**和工具集成进一步增强能力。

### 4. 实践中的优缺点
*   **优势：** 专家级的代码补全、代码理解和代码修复能力。
*   **局限性：** 
    *   **幻觉（Hallucinations）：** 可能生成不存在或过时的API。
    *   **上下文窗口限制：** 虽然可达100K-200K token，但并非所有token的影响力都一致。
    *   **延迟与成本：** 复杂任务响应可能需要数秒甚至数分钟，且高性能模型的token调用成本较高。

### 5. 课程安排与物流
*   **讲师背景：** Mihail Eric曾在亚马逊Alexa构建首批LLM，并创办过AI编程相关公司。
*   **交付物：** 包含9个每周作业和一个最终的开放式项目。
*   **亮点：** 课程将邀请顶尖AI开发工具初创公司的创始人进行客座讲座。
