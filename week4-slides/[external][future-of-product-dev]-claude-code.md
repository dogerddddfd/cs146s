这份PPT是由 Anthropic 的 Boris Cherny 带来的主题演讲，名为 **《欢迎来到 Claude Code》（Welcome to Claude Code）**。它探讨了在 AI 驱动下，编程范式正处于一个生产力指数级增长的**拐点**。

以下是本次内容的详细总结：

### 1. 行业趋势：编程的拐点
*   **生产力大爆发：** 编程语言和 IDE 的生产力正因 AI 而经历**指数级增长**。
*   **演进路径：** 从 1950 年代的打孔卡，到 1980 年代的 Smalltalk 和 Turbo Pascal，再到 2021 年的 Copilot，直到 2025 年出现的 **Claude Code**，工具的进化速度越来越快。
*   **验证手段的进化：** 验证（Verification）手段也从手动调试、自动化测试，进化到了 **AI 驱动的漏洞测试、单元测试和自我博弈（Self play）**。

### 2. Claude Code 的核心理念与定位
Claude Code 的设计目标是**“无处不在”**且**“深度集成”**：
*   **核心特性：** 终端原生（Terminal-native）、具备低级模型访问权限、且具有**无限的可扩展性（Hackable）**。
*   **全生命周期覆盖（SDLC）：** 它不仅能写代码，还贯穿软件开发的整个生命周期：
    *   **发现（Discover）：** 探索代码库历史、搜索文档。
    *   **设计（Design）：** 项目规划、制定技术规范、定义架构。
    *   **构建（Build）：** 实现代码、编写并执行测试、创建 PR。
    *   **部署（Deploy）：** 自动化 CI/CD、配置环境、管理部署。
    *   **支持与扩展（Support & Scale）：** 调试错误、大规模重构、监控性能。

### 3. 多样化的交付形式
Claude Code 不仅仅是一个命令行工具，它具有多种形态：
*   **终端（Terminal）：** 原生的交互界面。
*   **IDE 与 Web/iOS：** 深度集成在开发环境中，并提供移动端支持。
*   **GitHub App：** 通过 GitHub 直接指派任务（如 `@claude implement this`）。
*   **SDK：** 允许开发者将 Claude Code SDK 集成到自己的应用中，调用工具执行 Bash 命令或分析日志。

### 4. 典型使用场景与工作流
Claude Code 支持根据任务灵活调整工作流：
*   **代码库 Q&A：** 询问关于代码库的任何问题，如“为什么我们要在这个 PR 中添加这段逻辑？”。
*   **编写代码：** 包括单次生成（1-shot）、作为助手共同开发（Sidekick）或快速原型制作。
*   **工具集成：** 教授 Claude 使用你的自定义 CLI 工具或集成 MCP 服务器（如使用工具检查错误日志）。
*   **结构化工作流示例：**
    *   **发现 -> 规划 -> 确认 -> 编码 -> 提交**。
    *   **测试驱动：** 编写测试 -> 提交 -> 编码使测试通过 -> 迭代。
    *   **视觉驱动：** 实现代码 -> 截屏 -> 根据视觉反馈迭代直至符合原型。

### 5. 核心启示（Lessons）
Boris 总结了在 AI 时代开发产品的几条重要经验：
*   **面向未来建模：** 要**为六个月后的模型能力进行开发**，而不是局限于当前。
*   **保持进化：** 随时准备好跟随技术浪潮演进。
*   **“苦涩的教训”（The Bitter Lesson）：** 引用 Rich Sutton 的观点，强调从长远来看，**利用大规模计算能力**往往比利用人类的先验知识更有效。

**如何开始：**
开发者可以通过命令 `npm install -g @anthropic-ai/claude-code` 全局安装并体验。
